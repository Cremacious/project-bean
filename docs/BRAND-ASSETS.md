# Bedtime Quests — brand asset manifest (issue #87)

The complete, precise catalog of every image asset the whole app ships or renders,
plus the "drop in AI art and regenerate" workflow for a new logo.

WORKFLOW.md's [Brand assets](./WORKFLOW.md#brand-assets-logo--icons) section is the
short version. This is the full one. If they disagree, this file wins and WORKFLOW
should be updated.

## Guiding rules (do not break these)

1. **One shared art definition.** Every icon file is a derivation of the `INNER`
   SVG constant in [`scripts/gen-icons.ts`](../scripts/gen-icons.ts). Never
   hand-edit a generated file; change `INNER` and rerun `npm run gen:icons`.
2. **`<BrandMark>` stays byte-identical to `INNER`.** The inline React component
   in [`components/brand-mark.tsx`](../components/brand-mark.tsx) mirrors the
   same paths. Update both together, or the in-app mark drifts from the icons.
3. **Rounded vs square is a real distinction.** The web favicon and the in-app
   `<BrandMark>` use rounded corners (`rx=22` on a 0..100 square). The
   `apple-icon.png`, the App Store 1024, and the Play 512 are **full-bleed
   square (`rx=0`) with NO alpha channel**. iOS and Play apply their own corner
   mask and Apple rejects icons with transparency.
4. **Literal Paper Cut hex, not CSS vars.** Poppy `#FF6B4A`, Sun `#FFC24B`,
   Leaf `#2FB98A`, Cream `#FFF1DC`, Navy `#16283A`. This is what lets the
   raster files and the React component render identically everywhere.
5. **App-wide UI rule 1 applies to visible text on graphic assets.** No dashes
   as punctuation in any text baked into an image (feature graphic, hero,
   screenshots). Write "Ages 2 to 4", not "Ages 2-4".

## TL;DR for a logo refresh

1. Generate one **source master mark** (the new paper-boat-inspired logo).
   Ideally SVG. See [Source images you supply](#source-images-you-supply).
2. Paste the mark's SVG child elements into `INNER` in
   `scripts/gen-icons.ts` and into `<BrandMark>` in
   `components/brand-mark.tsx`. Keep them byte-identical.
3. Optionally generate a **hero illustration** for `app/welcome/page.tsx` and
   drop it at `public/brand/hero-welcome.webp` (see
   [Homepage hero](#homepage-hero-not-yet-wired)).
4. Run, in order:
   ```
   npm run gen:icons
   npm run gen:feature-graphic
   npm run gen:press-kit
   npm run build
   ```
5. Verify: browser favicon, `/` header mark, `/welcome` hero, `/press` downloads,
   `apps/mobile/assets/*.png` visually match the new art.

Full workflow with constraints and escape hatches is in
[AI drop-in workflow](#ai-drop-in-workflow).

## Implemented art — variant A2f (live in the pipeline)

The current logo is **variant A2f**: an origami paper boat with a pennant flag
sailing a green "sea of stars" toward a **sun-yellow crescent moon** (a disc with
a navy disc carved out by a `<mask>`) on deep-navy. The pristine source SVG lives
at [`public/brand/source/mark.svg`](../public/brand/source/mark.svg).

A2f changed how the shared art is structured, so a few of the older mechanics
below are superseded:

- **The art is split, not one flat string.** `scripts/gen-icons.ts` now exports
  `FOREGROUND` (boat + moon + star, all inside the x:8-92, y:8-92 safe area) and
  `WAVE` (the sea). `INNER` is kept as `FOREGROUND + WAVE` for back-compat.
- **The wave deliberately bleeds** to the tile edges and down to y:100. On
  rounded variants it is **clipped to the rounded corners** (`clipPath#tileF`);
  on full-bleed square variants (apple/store/iOS) it runs to the square edge with
  no clip. Do not pull the wave back inside the safe area.
- **Android adaptive is a true two-layer split** (see rows 11-12): the background
  layer is navy + the full-bleed wave; the foreground is boat + moon + star only,
  scaled 0.7 about center so nothing critical enters the launcher's circular crop.
- **`components/brand-mark.tsx` is now a Client Component** using `useId` to
  namespace the mask/clip ids per instance, because the header and footer both
  render `<BrandMark>` on one page and shared static ids would cross-wire.

To change the logo now, edit `FOREGROUND` / `WAVE` in `scripts/gen-icons.ts` and
mirror the same elements in `brand-mark.tsx` (keep the per-instance `useId`
wiring), then rerun the generators. The rest of the drop-in workflow is unchanged.

### AI-generation constraints (the checklist your prompts must satisfy)

Master mark:

- Square canvas, 2048x2048 or larger (or vector).
- Transparent background if raster.
- All essential art inside the central 64% (safe zone for Android adaptive icons).
- Nothing critical in the outer 8% (store icons round the corners).
- Paper Cut palette only: poppy `#FF6B4A`, sun `#FFC24B`, leaf `#2FB98A`, cream `#FFF1DC`, navy `#16283A`.
- No text baked into the mark. The product name lives in the feature graphic and hero, not the icon.
- No hairline detail. Test that the composition still reads at 16x16 (favicon size).
- "Simply inspired by" the current paper boat sailing to a crescent moon on a sea of stars. Refresh the execution, keep the motif.

Hero illustration:

- 2400x1600 (3:2), WebP primary, PNG fallback.
- Warm, high-contrast composition that reads on the `#16283A` navy panel.
- No dashes or em-dashes in any embedded text (UI rule 1). Prefer no text at all in the image, so we can lay copy over it.
- Same Paper Cut palette. Avoid greens or blues that clash with the leaf and navy tokens.

## AI drop-in workflow

The step-by-step "drop AI-generated art in, regenerate everything" flow.

### Step 0 - Prep

Confirm the source images satisfy the checklist in
[Source images you supply](#source-images-you-supply). Vector output beats
raster every time. If your AI tool cannot emit SVG, use the raster-only escape
hatch at the bottom of this section.

Create the source folder if it does not exist:

```
public/brand/source/
```

This folder is a staging area for the AI-generated masters. Nothing in
`public/brand/source/` is served to the browser or wired directly into the
build; the generators read from `INNER` (below), not from this folder. The
folder exists so you have a stable place to keep the master alongside the
brand set.

### Step 1 - Get the mark into `INNER`

`INNER` is a template literal of raw SVG child elements. It lives in
[`scripts/gen-icons.ts`](../scripts/gen-icons.ts) around lines 27-42. All 15
generated icon files render this string inside a navy square (either rounded
`rx=22` for the "rounded" variants or `rx=0` for the "square" full-bleed
variants).

1. Open the new mark SVG.
2. Copy every child element inside its outer `<svg>` tag. **Skip the outer
   `<svg>` wrapper and any background `<rect>` that fills the whole canvas** —
   the pipeline adds its own navy background rect.
3. Replace the body of the `INNER` template literal with those elements.
4. Do the same into the inline SVG in
   [`components/brand-mark.tsx`](../components/brand-mark.tsx) so the in-app
   mark stays byte-identical. Keep the outer `<rect width="100" height="100"
   rx="22" fill="#16283A" />` background rect that already lives in the
   component; replace only the child elements below it.

If the new mark introduces a very different silhouette, also update the `MONO`
constant a few lines below `INNER`. `MONO` is the white-on-transparent
Android 13+ themed layer: only the outer shape and a moon carve need to be
present, because the launcher tints it via the wallpaper.

**Do not** touch the `roundedSvg` / `squareSvg` / `safeZone` / `artSvg`
helpers. They are the platform adapters and stay put.

### Step 2 - Regenerate all derived assets

From the repo root, in order:

```
npm run gen:icons
npm run gen:feature-graphic
npm run gen:press-kit
```

- `gen:icons` rewrites the 15 files in the [icon-pipeline table above](#generated-assets-from-the-inner-art-via-npm-run-genicons).
- `gen:feature-graphic` rewrites the 2 files in the [feature-graphic table](#generated-assets-feature-graphic-pipeline-via-npm-run-genfeature-graphic). It imports the new `INNER` transitively, so it is automatically on-brand.
- `gen:press-kit` remirrors the 7 files in the [press-kit table](#generated-assets-press-kit-mirror-via-npm-run-genpress-kit).

You should not need to touch any generated file. If a diff shows up in one you
did not expect, that is the pipeline working; if you have hand-edited a
generated file, you now know.

### Step 3 - Add the homepage hero (optional but recommended)

Drop the hero at `public/brand/hero-welcome.webp` (primary) and, if you have
one, `public/brand/hero-welcome.png` (fallback). Then wire it into the hero
section of [`app/welcome/page.tsx`](../app/welcome/page.tsx). Around line 122
you will find the hero panel:

```tsx
<div className="relative overflow-hidden rounded-3xl bg-[#16283A] px-6 py-14 text-center sm:px-12 sm:py-20">
  <div className="mx-auto flex max-w-2xl flex-col items-center">
    <BrandMark size="lg" />
    ...
  </div>
</div>
```

Two options:

- **Mark plus background image.** Add a `next/image` element as an absolutely
  positioned background layer inside the outer `<div>`, and keep `<BrandMark
  size="lg" />` in front. Best if the hero art is a background pattern.
- **Hero image replaces the mark.** Swap `<BrandMark size="lg" />` for a
  `next/image` element sized to fit the panel. Best if the hero art is the
  focal element.

Follow app-wide UI rules: no dashes in any text overlay, high contrast on the
navy background, mobile-first and responsive.

### Step 4 - Open Graph and Twitter share cards

Nothing to do. `lib/og-image.tsx` reads
`public/brand/icon-rounded.svg` at request time, and Step 2 just regenerated
it. `/opengraph-image` and `/twitter-image` will pick up the new mark on the
next request.

If you want the share card to feature the new hero art instead of the mark,
edit `lib/og-image.tsx`: read the hero PNG the same way the mark is read
(`readFile` -> `base64` -> `<img src="data:...">`) and swap or layer it.

### Step 5 - Verify

- `npm run gen:icons` should print the regenerated file list without errors.
- `npm run build` should complete cleanly. Next will fail loudly if the SVG in
  `INNER` is malformed.
- In a browser:
  - Favicon appears in the tab and matches the new mark.
  - `/` (signed in) shows the new mark in the header and footer.
  - `/welcome` shows the new mark, hero, and reads well on mobile.
  - `/press` downloads work and preview correctly.
- Eyeball the mobile assets: open `apps/mobile/assets/icon.png`,
  `android-icon-foreground.png`, and `splash-icon.png` in an image viewer to
  confirm nothing is clipped by the safe zone or the full-bleed corners.

### Raster-only escape hatch (last resort)

If your AI tool only emits PNG and vectorizing (Adobe Illustrator > Image
Trace, Vector Magic, Recraft, etc.) is not an option today:

- Save the master mark as `public/brand/source/mark-2048.png` (transparent
  background, 2048x2048 or larger).
- The current `scripts/gen-icons.ts` does **not** read from
  `public/brand/source/`. To use a raster master you would need to add a
  companion script that skips the vector outputs (`app/icon.svg`,
  `public/brand/icon-square.svg`, `public/brand/icon-rounded.svg`) and only
  emits the raster derivatives, compositing the PNG onto a navy background.
- This deliberately creates raster / vector drift: `app/icon.svg`, the two
  vector sources in `public/brand/`, and `components/brand-mark.tsx` will
  still render the old paper-boat art until you vectorize and paste into
  `INNER`. Vectorize when possible.

This escape hatch is documented, not shipped. Adding the companion script is a
follow-up if the raster path is actually needed.

---

## What this issue does not do

- **Does not generate final art.** You supply the AI-generated master mark
  (and hero); this manifest tells you what specs it must meet, and the
  pipeline turns it into every derived file.
- **Does not commit any generated files.** Nothing here regenerates the icon
  set. The moment you run `npm run gen:icons` on the new `INNER`, every file
  in the [pipeline tables](#generated-assets-from-the-inner-art-via-npm-run-genicons)
  changes.
- **Does not modify `INNER`, the `<BrandMark>` component, or any generated
  asset.** The current paper-boat placeholder stays put until you deliberately
  replace it per [Step 1](#step-1---get-the-mark-into-inner).
